# The Project

WindSite is a decision engine for the planning stage of offshore wind turbine projects. Users define a project zone on a map, or let our site find suitable ones for them. The system analyses marine datasets, evaluates technical suitability, estimates energy production, and provides a structured regulatory overview. Instead of a static analysis, feasibility becomes a connected decision graph.

![WindSite Demo](./assets/windsitecomplexview.png)

We modeled various inputs and outputs as modular nodes. Seabed depth, seabed composition, wind, location, and environmental constraints. Deterministic calculations handle energy modeling, while the AI agent structures reasoning and reporting. This ensures transparency and explainability.

# Backend Development

I mainly worked on the data analysis, compression, and conversion scripts for the AI agent.

For the AI agent to analyse the seabed information, various large datasets with a size of 5GB+ had to be compressed into an easily readable form. These compressed files had to be analysed and used by the agent for feasibility reports. I programmed various scripts that were implemented into the agent for handling these tasks, allowing it to probe specific regions of interest for analysis.

# Frontend Development

For the frontend, we opted to use Lovable to generate the platform from a template. The platform itself was programmed in React. The platform includes the homepage, login, projects page, and main dashboard.

Much of the flow chart functionality itself was created by us. Various requirements such as being able to interactively connect nodes together, allow useable sliders and toggles, and the implementation of different node styles had to be implemented. We were inspired by n8n modular flow chart like interface.

We had a fully working and clean user interface by the end of the hackathon. We were complimented by many attendees for the clean and satisfying user interface that makes planning offshore wind turbine farms simpler and more efficient.

# Agent Implementation

We choose to use Anthropic Claude Sonnet 4.5 as our agent that would scrape information and use deep reasoning to generate complex and insightful feasibility reports, detailing the various advantages/disadvantages for specific site locations. Claude was given the Python analysis scripts to use for analysing GSI and INFOMAR datasets.

To make the program more efficient, we implemented a traffic light system, where users could use their previous analysis runs to "flash re-run". This prevented Claude credits from being wasted, as if a simulation was rerun, it would only use new values/parameters given and keep the context of the previous run, unless the user specified otherwise. This allowed the site and program to run more efficiently.
